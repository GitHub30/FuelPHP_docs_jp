貢献

導入

Fuelはコミュニティ駆動のフレームワークです。
そして、フレームワークをより良いものにしたいと考える誰からの貢献に対してもオープンです。
チュートリアルを書く事、パッケージをリリースする事, (Core本体, Orm, ドキュメント, etc..) Fuelの公式な部分への貢献することにより達成することができます。

バグの修正

When you encounter a bug or see one in the issues list that you can or have resolved, and you want to contribute it back to the framework. Use GitHub to send a pull request. Depending on which version you are running the branch you pull on. Follow the next couple of steps to when you request a pull:

  • 私の現在のバージョンはまだサポートされていますか？
    されていない場合は、サポートされているバージョンにアップグレードして、バグがまだ存在するかどうか参照してください。
  • ローカルリポジトリ上のバージョンをチェックアウト
    $ git checkout -b 1.1/develop origin/1.1/develop or $ git checkout -b 1.0/develop origin/1.0/develop
  • バグを修正し、ローカルリポジトリにコミット
    説明的でわかりやすいコミットメッセージを使用して、バグを修正し、ローカルリポジトリにコミットしてください。 $ git commit -m "This is my awesome description."
  • あなたのGitHubのアカウントにそれをpush
    $ git push
  • pull リクエストを送る
    リクエストされるpullは、あなたのbranch (もしくは、あなたのbranch以外)と同じbranchの上です。 1.0/develop branch のpullリクエストは、 1.0/developへ、1.1/develop のものは、 1.1/develop上に、送ってください。 あなたが正しいbranchにリクエストできるように、"コミットの更新範囲"のインタフェースを使用してください。
  • 名誉を待つ
    pullリクエストがマージされる際には、あなたの名前は、貢献者のリストの一部となるでしょう。

バグ無しへの貢献

バグ無しへの貢献は、新しい機能、最適化および他のすべての変化に対してです。これらの要求はすべて、最新の開発ブランチで行われている必要があります。 最新バージョンの開発ブランチで変更を行い、pullを要求してください。
