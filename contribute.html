バグの修正

バグを発見した場合、またそのバグを修正したコードを提供したい場合はGithubへpull requestを送ってください。
どのバージョンなのかによって、ブランチが違うので、確認してください。
それから、pull request したいステップを追ってください:

  • バグが発生をした、そのFuelのバージョンは、まだサポートされているものですか？
    もしサポートされていないものなら、バージョンアップした後にバグが存在するか確認をしてください。
  • チェックアウトしたあなたのローカルレポジトリのバージョンです。
    $ git checkout -b 1.1/develop origin/1.1/develop もしくは $ git checkout -b 1.0/develop origin/1.0/develop
  • バグを修正して、あなたのローカルレポジトリにコミットしてください。
    わかりやすいコミットメッセージをつけてください。 $ git commit -m "This is my awesome description."
  • それをあなたのGitHubアカウントへ、pushしてください。
    $ git push
  • pull Requestを送信してください。
    Request the pull on the same branch as you are on (or you based your branch off). A 1.0/develop branch request pulls on 1.0/develop, 1.1/develop on 1.1/develop and so on. Use the "update commit range" interface to direct you request to the correct branch.
  • Wait for your moment of fame
    As soon as your request is merged you'll be part of the contributors list of Fame.

バグ解決への貢献

Non-bug contributions are new features, optimizations and all other alterations. All of these request must be made on the latest development branch. So pull the latest version's dev branch, make the changes and request a pull.
